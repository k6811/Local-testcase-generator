# SOP: Testcase Generator – Standard Operating Procedure

## Goal
Accept a user's feature/requirement description and return a structured set of test cases in Markdown format, generated by the local `llama3.2` model via Ollama.

---

## Inputs
| Field | Type | Source | Validation |
|---|---|---|---|
| `user_input` | `string` | Frontend Chat UI (POST body) | Must be non-empty |

## Outputs
| Field | Type | Destination | Format |
|---|---|---|---|
| `test_cases` | `string` | Frontend Chat UI (JSON response) | Markdown |

---

## Architectural Flow

```
[User] → types requirement in Chat UI
       ↓
[Frontend: index.html + script.js]
       ↓  POST /api/generate_testcases  { user_input: "..." }
[Layer 2: Navigation – FastAPI router in main.py]
       ↓  validates input, wraps with PROMPT_TEMPLATE
[Layer 3: Tool – ollama_client.py]
       ↓  POST http://localhost:11434/api/generate
[Ollama: llama3.2 model]
       ↓  returns generated text
[Layer 2: Navigation – FastAPI router]
       ↓  returns { test_cases: "..." }
[Frontend: renders Markdown in chat bubble]
       ↓
[User] ← sees structured test cases
```

---

## Tool Logic (Layer 3)
- **File:** `tools/ollama_client.py`
- **Input:** Full final prompt (string)
- **Action:** HTTP POST to Ollama API with `model`, `prompt`, and `stream: false`
- **Output:** Raw response string from Ollama
- **Edge Cases:**
  - Ollama not running → Raise `ConnectionError` with descriptive message
  - Empty response from Ollama → Return placeholder string `"[No output generated]"`
  - HTTP error codes → Raise `RuntimeError` with status code

---

## Prompt Template Rules
- Template is stored in `backend/prompt_template.txt`
- Template must contain the literal token `{user_input}` for injection
- **NEVER** modify the template at runtime
- Template is loaded once at application startup

---

## Edge Cases & Constraints
| Scenario | Expected Behaviour |
|---|---|
| Empty user input | HTTP 400 – "User input cannot be empty" |
| Ollama not reachable | HTTP 500 – "Failed to communicate with Ollama: ..." |
| Model not found | HTTP 500 – Ollama error surfaced to user |
| Extremely long input | Accepted as-is; Ollama handles context window limits |

---

## Golden Rule
> **If the prompt logic or flow changes, update this SOP BEFORE updating the code.**
